# Public Screenshots Manifest

This folder is for redacted, publishable screenshots only.

Do not place raw customer screenshots, private IPs, passwords, tokens, `.env` output, or unreviewed logs here.

## Expected Files

Add these files when capturing a public case study:

- `01-diagnostics-run.png`
- `02-report-folder.png`
- `03-container-before.png`
- `04-healthcheck-failed.png`
- `05-fix-applied.png`
- `06-healthcheck-passed.png`
- `07-backup-created.png`
- `08-checksum-verified.png`
- `09-restore-test.png`
- `10-monitoring-dashboard.png`
- `11-incident-report.png`

## Capture Rules

- show the command and the result when possible
- crop unrelated desktop clutter
- redact domains or IPs if needed
- prefer terminal zoom that is readable in a GitHub README or portfolio page
- review logs before publishing

## How To Use This Folder

1. Keep raw captures in a private location outside Git.
2. Export redacted versions into this folder.
3. Use `captions.md` to keep the case-study text consistent.
4. Link the final case study from `README.md`, the portfolio, and the GitHub profile hub.
