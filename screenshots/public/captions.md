# Public Screenshot Captions

Use these short captions in the case study or portfolio.

## Suggested Captions

- `01-diagnostics-run.png`  
  Initial diagnostics run on the target host before changes.

- `02-report-folder.png`  
  Generated report folder with collected diagnostics output.

- `03-container-before.png`  
  Container state before the fix, showing the failing or suspicious service.

- `04-healthcheck-failed.png`  
  Public endpoint check failing before remediation.

- `05-fix-applied.png`  
  Scoped configuration or command change used to address the issue.

- `06-healthcheck-passed.png`  
  Endpoint check succeeding after the fix.

- `07-backup-created.png`  
  Backup archive created for the scoped app or data volume.

- `08-checksum-verified.png`  
  Integrity verification of the backup artifact.

- `09-restore-test.png`  
  Restore test completed in a disposable target.

- `10-monitoring-dashboard.png`  
  Lightweight monitoring view showing the service in a known-good state.

- `11-incident-report.png`  
  Final client-facing incident report or handover note.

## Writing Rule

Captions should describe the operational outcome, not just the tool name.
